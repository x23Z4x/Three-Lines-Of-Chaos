# Three-Lines-Of-Chaos
Three lines of Python. Endless chaos. A simple process-spawning experiment exploring uncontrolled process creation. Don't run it on your main machine.

======================================================================================

# Process Spawn Experiment

This repository demonstrates a simple programming concept: what can happen when a program continuously creates new instances of itself without any limits.

# Educational Purpose

The goal of this experiment is to understand:

- Process creation in operating systems
- Resource exhaustion (CPU, RAM, and process limits)
- Why software should implement proper safeguards
- The importance of testing code in isolated environments

# Warning

Running uncontrolled process-spawning code may cause system instability, unresponsiveness, or data loss.

Always use virtual machines or disposable test environments when experimenting with resource-intensive code.

# What You Can Learn

- How operating systems manage processes
- Why infinite loops can be dangerous
- Basic defensive programming practices
- System resource monitoring and troubleshooting
